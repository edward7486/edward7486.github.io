Background and Problem

Datalink is a product focused on "data ingestion", allowing customers to sync various financial data sources they have directly into Apptio's platform. The Datalink user interface (UI) needed an update, due to:

  • The UI not matching other Apptio products
  • Minor usability issues
  • A consensus of dislike for the UI from within the company

Project Goals

Redo the UI to match existing Apptio products, uncover and fix existing usability issues.

My Role

I was the sole designer and front-end developer for this project, having guidance from my manager, and a software engineer on the Datalink team.

Research and Exploration

I first spent time learning about Datalink and discovered the product was simple to use, but had some usability issues such as user deletion with no confirmation dialogue (violates Nielsen's error prevention heuristic). I made note of these issues and began speaking with engineering to learn about the user and use cases. I discovered users don't commonly spend more than one day per month using Datalink, and a typical scenario of use is when the user is under duress because of a data ingestion issue.

Sketches

I started the design process by getting my ideas out on paper. To accomplish a refresh I wanted to make better use of screen space, change the visual hierarchy to top-to-bottom to match other Apptio products, and provide solutions to small UI nuances like the action buttons. To aid in sketching, I referred to a sitemap I created to ensure I was capturing all content and actions appropriately.

Iteration #1

Once I received feedback and approval internally, I began rapid prototyping with Bootstrap for demo and testing purposes. To make the prototype as life-like as possible, I added jQuery form validation (forms are a large part of Datalink).

Usability Fixes

A major usability fix was adding a confirmation dialogue for deletion of users. This was accomplished with a Bootstrap modal notifying users about the consequences of deleting a user and asking for confirmation. To design for the established typical user scenario, I ensured that users were directed to the correct default page when entering Datalink administration.

Iteration #2

The navigation bar was revised because the navigation used did not fit the needs of the more simple Datalink product (breadcrumbs were not needed and titles seemed redundant). I switched out the navigation bar for a smaller bar, and drastically simplified the navigation to only show necessary components: logo, title of page, primary dropdown menu, and a pulled right hamburger menu for help and logging out.

What I Learned

Datalink was an amazing project that kept me challenged and learning everyday. What I learned from this project was to be fearless - that most technical challenges can be overcome with sheer perseverance and hard work. I also learned how to set expecations with stakeholders, by setting and creating my own project deliverable timeline. I learned how to work in an Agile software development lifecycle. Lastly, I learned the value of getting constant feedback during the design process.
